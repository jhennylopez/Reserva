MANUAL DE USUARIO: SISTEMA DE GESTIÓN DE RESERVAS RommyEC

Este manual proporciona las instrucciones necesarias para operar el sistema desarrollado por Luis Franklin Salazar Poma. El software utiliza una arquitectura de 4 capas para garantizar la integridad de los datos y una experiencia de usuario fluida.

1. Inicio de Sesión y Seguridad

El sistema cuenta con un control de acceso dual para proteger la información y delimitar funciones.

  • Identificación: Ingrese su correo electrónico institucional o personal registrado y su contraseña secreta.
  • Roles de Usuario:
    • Administrador: Tiene control total sobre los catálogos de huéspedes, alojamientos y la gestión de todas las reservas.
    • Huésped: Perfil restringido que solo permite visualizar disponibilidad y gestionar sus propias reservaciones.
  • Cierre de Sesión: Siempre utilice la opción Archivo > Cerrar Sesión para asegurar que sus datos no queden expuestos en terminales públicas.

2. Gestión de Huéspedes (Módulo Administrativo)

Este módulo permite mantener actualizada la base de datos de clientes que harán uso de los servicios.

2.1 Registro de Nuevos Huéspedes

  1. Diríjase a Huésped > Registrar.
  2. Cédula: Ingrese los 10 dígitos obligatorios (el sistema validará automáticamente que no existan letras).
  3. Datos Personales: Complete nombres, apellidos, correo y teléfono.
  4. Presione Guardar. El sistema le confirmará si el registro fue exitoso.

2.2 Búsqueda y Actualización

Si un cliente cambia de teléfono o correo, utilice Huésped > Actualizar. Deberá ingresar la cédula para cargar los datos actuales, realizar el cambio y presionar Actualizar.

3. Catálogo de Alojamientos

Aquí se administran las propiedades, habitaciones o espacios disponibles para reserva.

  • Capacidad Máxima: Es el dato más crítico. El sistema impedirá realizar reservas que superen este número de personas para evitar sobrecupos.
  • Ubicación y Descripción: Proporcione detalles claros (ej. "Cabaña rústica con vista al volcán") para facilitar la elección del cliente.

4. Proceso de Reservaciones

Es el corazón del sistema y donde ocurre la validación lógica más importante.

4.1 Cómo realizar una Nueva Reserva

  1. Seleccione Reservas > Nueva Reserva.
  2. Alojamiento: Elija el ID del alojamiento disponible. El sistema mostrará la capacidad máxima.
  3. Fechas: Utilice los calendarios para marcar la Fecha de Entrada y Fecha de Salida.
    • Nota: El sistema no permite que la fecha de salida sea anterior a la de entrada.
  4. Cantidad de Personas: Ingrese cuántos huéspedes ocuparán el lugar. Si el número supera la capacidad del alojamiento, el sistema emitirá una alerta y bloqueará el registro.
  5. Confirmar: Presione Registrar.

4.2 Cancelación de Reservas

En caso de que un cliente desista, ingrese a Reservas > Cancelar. Seleccione el ID de la reserva de la lista inferior y presione Eliminar. Se le pedirá una confirmación final para evitar pérdidas accidentales de datos.

5. Solución de Problemas (FAQ)

Problema

Causa Probable

Solución

Error "CS1061" al compilar

Desconexión entre el diseño y el código.

El desarrollador debe limpiar las referencias en el archivo .Designer.cs.

El botón Buscar no responde

El evento no está enlazado en el Inspector de Propiedades.

Verifique que el evento Click apunte al método correcto.

No puedo registrar más personas

Se ha superado el límite físico del alojamiento.

Seleccione un alojamiento con mayor capacidad o divida la reserva en dos habitaciones.

Para reportar fallas técnicas en la base de datos SQL Server o errores de desbordamiento de datos, contacte al administrador del sistema o al departamento de desarrollo asociado a la ESPOCH.
